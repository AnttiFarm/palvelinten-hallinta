Karvinen 2026: Hello Ansible:

-Ansiblen kaikki toiminta tapahtuu SSH:n kautta, eikä orjapalvelimilla tarvitse asentaa erillistä ohjelmaa.

-Ansiblen konfiguraatiota varten tehdään sille oma hakemisto, johon tehdään hosts.ini tiedosto, missä on lueteltuna hallittavat hostit.

-Playbookit määrittelevät mitä tehtävää suoritetaan millä hostilla.

-Rooleja voi olla monia, ja näillä rooleilla on omat alihakemistonsa, joissa on esim. niiden tehtävät.
## a)
Asensin ssh:n virtuaalikoneelle puhtaaseen debian asennukseen. Käytin komentoja sudo apt-get install ssh ja ssh localhost uptime joka antoi koneen päälläolo ajan, eli ssh toimi.
## b)
Sain luotua julkisen avaimen komennolla ssh-keygen ja kopioin sen ssh-copy-id localhost. Yhdistin ssh localhost komennolla eikä se pyytänyt enää salasanaa.
## c)
Asensin ansiblen komennolla sudo apt-get install ansible micro bash-completion tree. Tässä mukana tuli muitakin käteviä ohjelmia. Seuraavaksi tein ansiblelle oman hakemiston omassa kotihakemistossani mkdir ansible komennolla. Tähän hakemistoon tein hosts.ini tiedoston johon lisäsin localhost. Seuraavaksi testasin että se toimi komennolla ansible all -a 'uptime' -i hosts.ini. Tässä kohtaa se valitti python versiosta vaikka siinä ei ollut oikeaa ongelmaa, poistin tämän lisäämällä hosts.ini tiedoston pohjalle [all:vars] ansible_python_interpreter=/usr/bin/python3. Testasin uudelleen ja varoitusta ei tullut. Seuraavaksi tein ansible.cfg tiedoston johon lisäsin [defaults] inventory = hosts.ini. Tämän avulla ei tarvitse kirjoittaa -i hosts.ini joka kerta ansible komentojen loppuun. Sitten tein sites.yml tiedoston johon kirjoitin

- hosts: all
  roles:
    - hello

eli jokainen host saa roolin hello. Testasin tätä komennolla ansible-playbook site.yml. Tästä tuli virheilmoitus, mutta itse site.yml on luettu, roolia hello ei löytynyt. Joten tein roolin komennolla mkdir -p roles/hello/tasks/. Tähän hakemistoon lisäsin main.yml johon kirjoitin

- copy:
    dest: /tmp/hello-ansible
    content: "Hello this is Antti"
  
Suoritin tämän ansible-playbook site.yml. Ohjelma sanoi että yksi asia oli muuttunut. Tarkistin tämän ssh localhost 'cat /tmp/hello-ansible'. Tämä tulosti Hello this is Antti. Eli kaikki toimi kuten pitikin. Lisäsin vielä ansible.cfg tiedoston loppuun display_args_to_stdout = true, jolla pystyy nähdä mitä ansible on tehnyt ilman erillistä tarkistusta.
