### Chacon and Straub 2014: Pro Git, 2ed: 1.3 Getting Started - What is Git?

- Git tallentaa projektin tilan ja tallentaa siihen liittyvän viitteen.
- Jos mitään ei ole muuttunut Git ei tallenna tieodostoja uudelleen vaan linkkaa viimeisimpään identtiseen tiedostoon.
- Suurin osa projektin sisällöstä tallenttuu paikallisesti: oma kopio.
  - Nopeampaa hallintaa.
  - Pystyy tehdä muutoksia ilman internet yhetyttä.
- Mitään projektin sisällä ei pysty muuttamaan ilman että Git tietäisi siitä.
- Git käyttää kolmea tilaa tiedostoille:
  - Modified, tämä ei ole vielä tietokannassa.
  - Staged, tämä on menossa tietokantaan.
  - Commited on tietokannassa.
Working tree on mitä voi muokata, staging area on tiedosto jossa lukee mitä muutoksia tuodaan Git hakemistoon.

### `git add --all && git commit; git pull && git push` komennon jokainen osa

git osa kertoo että kyseessä on git komento, add lisää muutokset "staged" tilaan ja --all tarkoittaa että kaikki muutokset kaikkiin tiedostoihin viedään staged tilaan. && oikealla olevat komennot suoritetaan vain jos vasen puoli onnistuu.
git commit tallentaa projektin tilan. git pull hakee kaikki muutokset git hakemistosta omaan paikalliseen kopioon ja git push työntää kaikki "commit" muutokset git hakemistoon.
